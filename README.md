# Build

Test the production release build, by compiling the files with:

    npm run release

And then serving the files with the Shadow-cljs builtin webserver:

    npx shadow-cljs server


# Architecture

When using `pdf.js` as a node library dependency,
Shadow-cljs builds with a browser target would fail
complaining that dependencies like `os` and `fs` wasn't met.

Instead `pdf.js` was live loaded via a CDN and not builtin with the webapplication.
